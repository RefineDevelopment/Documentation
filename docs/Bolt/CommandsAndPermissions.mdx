# Bolt Practice Plugin - Commands and Permissions

## Commands

### Main Commands

| Command | Description | Permission |
| :--- | :--- | :--- |
| `/bolt` | View bolt commands (admin) or info (player) | - |
| `/bolt info` | View information about Bolt | - |
| `/bolt reload` | Reload bolt's configuration files | `bolt.admin.reload` |
| `/bolt theme <color>` | Set bolt's theme color | `bolt.admin.theme` |
| `/bolt save` | Save bolt's configuration files | `bolt.admin.save` |
| `/bolt update` | Force update leaderboards | `bolt.admin.leaderboard` |
| `/bolt checkArenas` | Scan arenas to find broken/invalid arena | `bolt.admin.arenas` |
| `/bolt reset` | Reset the Bolt's DB Collections (Console only) | `bolt.admin.reset` |
| `/bolt setspawn` | Set spawn location | `bolt.admin.setup` |
| `/bolt goldenHead` | Receive a pre-made Golden Head | `bolt.admin.setup` |
| `/bolt instantHead` | Receive a pre-made InstantUHC Golden Head | `bolt.admin.setup` |
| `/bolt fireball` | Receive a pre-made Fireball | `bolt.admin.setup` |
| `/bolt rename <name>` | Rename the item in your hand | `bolt.admin.setup` |
| `/bolt unbreakable` | Mark the item in your hand as unbreakable | `bolt.admin.setup` |
| `/bolt effect <type> <lvl>` | Give yourself an infinite potion effect | `bolt.admin.setup` |
| `/bolt refill` | Refill your Inventory with Soup or Potions quietly | `bolt.admin.refill` |
| `/bolt forfeit [target]` | Force end someone's match | `bolt.admin.forfeit` |
| `/bolt resetStats <target>` | Reset a target's profile | `bolt.admin.resetprofile` |
| `/bolt head <target>` | Get a player's head | `bolt.admin.head` |
| `/bolt debugPaste` | Debug paste a schematic at your location | `bolt.admin.debugpaste` |
| `/bolt developerDebug` | Developer only command to toggle internal debugging | (Hidden) |

## Notes

1. Most commands check for player state (in lobby, in match, etc.) before execution
2. Many commands have built-in cooldowns or rate limiting
3. Permission checks are done both via `@Requires` annotation and manual `hasPermission()` checks
4. Some permissions are dynamic (e.g., cosmetic permissions follow pattern `bolt.<type>.<name>`)
5. Console-only commands are marked and will fail if executed by a player
6. Some commands have hidden flags (like `developerDebug`) that don't appear in help menus
